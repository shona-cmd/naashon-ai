# Contributing to AI Coding Assistant

Thank you for your interest in contributing! Here's how you can help:

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/ai-coding-assistant.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Install dependencies: `npm install`

## Development Setup

```bash
# Install dependencies
npm install

# Build the project
npm run build

# Watch for changes
npm run watch

# Run linter
npm run lint

# Run tests
npm test

# Launch extension in debug mode
# Press F5 in VS Code
```

## Code Style

- Use TypeScript for all code
- Follow the ESLint configuration
- Use meaningful variable and function names
- Add comments for complex logic
- Write tests for new features

## Committing Changes

1. Commit with clear, descriptive messages
2. Reference issues when applicable (e.g., "Fixes #123")
3. Follow conventional commit format:
   - `feat:` for new features
   - `fix:` for bug fixes
   - `docs:` for documentation
   - `test:` for tests
   - `chore:` for build/tooling changes

## Testing

- Write unit tests for new functions
- Ensure all tests pass: `npm test`
- Check code coverage
- Test in debug mode with the extension

## Pull Request Process

1. Update README.md with any new features
2. Update CHANGELOG.md with your changes
3. Ensure all tests pass and linting is clean
4. Provide a clear description of your changes
5. Link related issues

## Reporting Bugs

Use the GitHub Issues to report bugs. Include:
- Clear description of the issue
- Steps to reproduce
- Expected vs actual behavior
- VS Code version and extension version
- Error messages or logs

## Feature Requests

Suggestions are welcome! Please create an issue with:
- Clear description of the feature
- Use cases
- Examples of how it would work

## Questions?

Feel free to open an issue or reach out to the maintainers!
